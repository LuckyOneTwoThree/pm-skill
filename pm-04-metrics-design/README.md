# 模块5：产品度量设计（开发前）

## 定位

承接PRD阶段的产出，在开发启动前设计完整的度量体系，确保产品上线后能够用数据驱动决策。目标是**用数据减少决策中的猜测，而非用数据为决策做背书**。

## 何时使用

- PRD已定稿，需要在开发前规划度量体系
- 需要设计指标体系、埋点方案和Dashboard
- 需要确保北极星指标与OKR对齐
- 需要为上线后的数据驱动决策做好准备

## 子模块与编排器

| 子模块 | 编排器 | 作用 | 何时调用 |
|--------|--------|------|----------|
| 产品度量设计 | metrics-orchestrator | 设计指标体系、埋点方案和Dashboard | 需要在开发前建立度量体系时 |

## Pipeline Skill 清单

### 产品度量设计（3个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| metrics-system | 设计指标体系，推荐北极星指标候选，建立层级指标（北极星→驱动→辅助） | OKR、PRD、北极星指标 | metrics-system.json |
| tracking-plan | 生成埋点方案，覆盖事件定义、属性、触发条件 | 指标体系、用户流程 | tracking-plan.json |
| metrics-dashboard | 配置Dashboard布局和告警阈值 | 指标体系、埋点方案 | dashboard.json |

## 执行顺序

```
阶段1            阶段2            阶段3
┌──────────┐  ┌──────────┐  ┌──────────────┐
│ 指标体系  │→│ 埋点方案  │→│ Dashboard    │
│ 设计     │  │ 设计      │  │ 配置         │
└──────────┘  └──────────┘  └──────────────┘
```

- 指标体系设计是起点，确定北极星和层级指标
- 埋点方案依赖指标体系，确保每个指标可采集
- Dashboard配置依赖指标体系和埋点方案，确保可视化呈现
- metrics-dashboard 是本模块的最终产出

## 输出路径

```
output/pm-metrics-design/
├── metrics-system/
├── tracking-plan/
└── metrics-dashboard/
```

## 阶段卡口

进入开发阶段前需满足：

- 指标体系完成：北极星指标人类已选择
- 埋点方案完成：埋点方案人类已审核
- Dashboard完成：Dashboard布局人类已确认

## 人类决策点

| 决策点 | 说明 |
|--------|------|
| 北极星指标选择 | AI推荐3个候选北极星指标，人类选择最终指标 |
| 埋点方案审核 | AI生成埋点方案，人类审核业务逻辑和隐私合规 |
| Dashboard布局确认 | AI配置Dashboard，人类确认布局和告警阈值 |

## 核心信念

- 用数据减少决策中的猜测，而非用数据为决策做背书
