# 产品提案书自动生成

## 核心原则

1. **一页纸决策**——决策层没有时间看长文，核心论点必须一页纸说清
2. **数据支撑主张**——每个主张必须有数据或证据支撑，禁止空洞判断
3. **风险前置**——主动暴露风险比隐藏风险更负责任
4. **可否决**——提案的目的是让决策者做出"做或不做"的判断，不是说服

## 交互模式

🤖→👤 AI建议人类审批

## 执行步骤

### Step 1: 机会陈述

用一句话说清"为什么是现在"：

**机会陈述模板**：
```
{目标用户} 在 {场景} 下，面临 {核心痛点}，
现有方案 {不足}，
我们通过 {差异化方式}，
预期能 {量化收益}。
```

**数据支撑**：
- 市场规模（来自 TAM/SAM/SOM）
- 用户痛点验证（来自用户研究报告）
- 竞品空白（来自竞品分析报告）

### Step 2: 产品概述

**产品一句话定义**：
```
{产品名} 是一款面向 {目标用户} 的 {品类} 产品，
通过 {核心能力} 解决 {核心痛点}，
与 {主要竞品} 的区别是 {差异化}。
```

**核心功能清单**（MVP范围）：

| 功能 | 用户价值 | 优先级 | 预估工期 |
|------|---------|--------|---------|
| | | Must/Should/Could | |

**非目标（明确不做什么）**：
- 不做 xxx，因为 yyy
- 不做 xxx，因为 yyy

### Step 6: 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 市场风险：需求不如预期 | | 高 | MVP快速验证，3个月内判断 |
| 竞争风险：巨头入场 | | 高 | 差异化壁垒构建 |
| 技术风险：核心能力实现难度 | | 中 | 技术预研先行 |
| 资源风险：人力不足 | | 中 | 分阶段投入，MVP最小团队 |
| 合规风险：数据/隐私 | | 中 | 上线前合规评估 |

### Step 7: 决策建议

| 选项 | 说明 | 推荐度 |

| ❌ 否决 | 不符合当前战略方向 | — |

**下一步行动**（批准后）：
1. 确认核心团队
2. 启动PRD编写
3. 技术预研启动

## 决策规则

| 条件 | 决策 |
|------|------|
| TAM数据缺失 | 基于品类和AI知识估算，标注"缺乏TAM数据，建议补充" |
| 用户研究数据缺失 | 痛点基于假设，标注"缺乏用户研究验证" |
| 竞品数据缺失 | 竞争格局基于AI知识，标注"建议补充竞品分析" |
| 财务数据全部缺失 | 使用行业基准值，标注"使用行业基准，需根据实际情况调整" |
| 资源约束未提供 | 按标准团队配置估算，标注"需确认实际资源" |

## 质量检查

- [ ] 一页纸摘要完整（机会+产品+投入+建议）
- [ ] 机会陈述有数据支撑
- [ ] 核心功能不超过5个
- [ ] 非目标至少2条
- [ ] 竞品至少3个
- [ ] 单位经济LTV/CAC已计算
- [ ] 风险至少4条
- [ ] 决策建议明确

## 降级策略

| 缺失的上游输入 | 降级方案 | 输出影响 |
|---------------|---------|---------|
| 竞品报告缺失 | 基于AI知识推导竞争格局 | 竞争分析可能不够精准 |
| TAM数据缺失 | 基于品类估算市场规模 | 市场规模为估算值 |
| 用户研究报告缺失 | 痛点基于假设和AI知识 | 缺乏实证支撑 |
| 机会简报缺失 | 机会评估基于其他数据推导 | 机会评分可能不够精确 |
| 定位陈述缺失 | 差异化定位基于竞品分析推导 | 定位可能不够清晰 |
| 产品名称与品类（用户提供） | 若用户未提供产品名称与品类，提示用户提供或跳过该输入相关步骤 | — |
| 商业目标（用户提供） | 若用户未提供商业目标，提示用户提供或跳过该输入相关步骤 | — |
| 资源约束（用户提供） | 若用户未提供资源约束，提示用户提供或跳过该输入相关步骤 | — |
