# Pipeline 5: 价值曲线自动绘制

## 核心原则

1. **选项生成优于单一推荐**：每个关键决策点生成2-3个可比较选项，由人类选择而非AI替选
2. **数据驱动填充人类驱动选择**：AI负责数据整合与逻辑推导，人类负责方向判断与最终决策
3. **假设显式化**：所有推断内容必须标注为假设，包含风险等级和验证方法
4. **财务建模自动化**：单位经济、敏感性分析等财务计算由AI自动完成，人类只审核结论

## 交互模式
🤖→👤 AI建议人类审批

## 输入

| 输入项 | 类型 | 必填 | 来源 | 说明 |
|--------|------|------|------|------|
| 竞品分析数据 | JSON | 是 | output/pm-discovery/market-competitor-intel/competitor-intel.json | 竞品功能对比、市场定位 |
| 自身产品能力评估 | JSON | 是 | 用户提供 | 产品功能矩阵、成熟度评估 |
| 用户研究数据 | JSON | 是 | output/pm-discovery/user-research-user-modeling/persona.json | 用户关注要素、痛点优先级 |

## 执行步骤

### Step 1: 竞争要素提取
从输入数据中提取Top 8-12个竞争要素：
- 覆盖功能、性能、体验、生态等维度
- 参考用户研究中的用户关注权重
- 确保要素可量化评分

### Step 2: 各方评分
对每个竞争要素进行1-5分评分：

| 评分 | 含义 |
|------|------|
| 1分 | 劣于行业标准 |
| 2分 | 达到行业标准 |
| 3分 | 优于行业标准 |
| 4分 | 显著优于行业标准 |
| 5分 | 行业领先/独占 |

评分对象：
- 我方产品
- 主要竞品1
- 主要竞品2
- ...

### Step 3: 蓝海动作识别
应用蓝海战略四动作框架：

| 动作类型 | 定义 |
|----------|------|
| **Eliminate** | 消除哪些行业习以为常但实际价值低的要素 |
| **Reduce** | 降低哪些要素到行业标准以下 |
| **Raise** | 提高哪些要素到行业标准以上 |
| **Create** | 创造哪些行业从未提供的要素 |

### Step 4: 差异化强度评估
计算差异化强度评分：
- 面积法：我方价值曲线与竞品围合面积
- 归一化处理，输出0-1之间的评分

## 输出

**存储路径**：`output/pm-strategy/positioning-value-curve/value-curve.json`

**输出文件**：value-curve.json

```json
{
  "competition_factors": [
    {

      "scores": {
        "self": 4,
        "competitor_1": 3,
        "competitor_2": 2
      },
      "user_attention": "high",
      "blue_ocean_action": "Raise"
    }
  ],
  "differentiation_strength": 0.72,
  "blue_ocean_summary": {
    "eliminate": ["要素1", "要素2"],
    "reduce": ["要素3"],
    "raise": ["要素4"],
    "create": ["要素5"]
  }
}
```

## 决策规则

| 情况 | 处理方式 |
|------|----------|
| 差异化强度 ≥ 0.5 | 正常通过 |
| 差异化强度 < 0.5 | 警告，提示需加强差异化 |
| 蓝海动作建议 | 需人类审核战略意图 |

## 质量检查

1. **数据支撑**：竞争要素来源有数据依据（用户研究/竞品分析）
2. **逻辑依据**：蓝海动作有合理逻辑支撑
3. **可执行性**：差异化要素可转化为产品路线

## 降级策略

当上游文件不存在时，本Skill仍可独立执行：

| 缺失的上游文件 | 降级方案 |
|---------------|---------|
| competitor-intel.json | 用户提供竞品列表和自身特点 → 基于AI知识绘制价值曲线，标注"缺乏竞品情报数据" |
| 自身能力评估数据 | 用户提供自身特点描述 → 基于描述评分，标注"缺乏结构化能力评估" |
| competitor-intel.json + 自身能力评估 | 用户提供竞品列表和自身特点 → 绘制价值曲线，整体置信度降低 |
| 所有上游文件均缺失 | 提示用户先执行前序阶段，或基于用户提供的竞品列表和自身特点绘制价值曲线 |

数据获取说明：
- 本Skill需要竞品分析和自身能力评估数据，请通过以下方式之一提供：
  1. 直接描述自身产品特点和主要竞品
  2. 上传competitor-intel.json文件
  3. 提供数据文件路径
- AI不负责外部数据采集，仅负责分析
