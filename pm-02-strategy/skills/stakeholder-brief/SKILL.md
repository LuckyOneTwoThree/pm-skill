# Pipeline 14: 战略简报自动生成

## 核心原则

1. **选项生成优于单一推荐**：每个关键决策点生成2-3个可比较选项，由人类选择而非AI替选
2. **数据驱动填充人类驱动选择**：AI负责数据整合与逻辑推导，人类负责方向判断与最终决策
3. **假设显式化**：所有推断内容必须标注为假设，包含风险等级和验证方法
4. **财务建模自动化**：单位经济、敏感性分析等财务计算由AI自动完成，人类只审核结论

## 交互模式

🤖→👤 AI建议人类审批

## 输入

| 输入项 | 类型 | 必填 | 来源 | 说明 |
|--------|------|------|------|------|
| 产品战略文档 | markdown | 是 | output/pm-strategy/stakeholder-strategy-doc/strategy-doc.md | Pipeline 13 输出的完整战略文档 |

## 执行步骤

1. **读取战略文档**：从 strategy-doc.md 提取核心战略要素
2. **生成一句话战略方向**：用一句话概括产品战略的核心
3. **分析市场时机**：提炼"为什么是现在"的关键论据
4. **提炼差异化定位**：总结"我们怎么赢"的核心逻辑
5. **摘要OKR**：提取核心目标和关键结果
6. **整理风险假设**：筛选最高影响的风险和假设
7. **汇总资源需求**：整理需要的支持
8. **格式适配**：根据目标渠道调整格式

## 输出

**存储路径**：`output/pm-strategy/stakeholder-brief/`

**输出文件**：stakeholder-brief.md

### 一句话战略方向

用一句话概括产品战略的核心，格式：

> **通过[方式]，为[目标用户]解决[问题]，实现[目标]。**

### 为什么是现在

**市场时机分析**:
- 市场窗口期
- 竞争态势
- 用户需求成熟度
- 技术/政策环境

### 我们怎么赢

**差异化定位**:
- 核心竞争优势
- 独特价值主张
- 关键成功因素

### 成功长什么样

**OKR摘要**:
- 核心目标（O）
- 关键结果（KR）及其指标

### 关键风险和假设

**风险假设矩阵**:

| 风险/假设 | 影响 | 应对策略 |
|-----------|------|----------|
| 风险1 | 高/中/低 | 缓解措施 |
| 假设1 | 高/中/低 | 验证计划 |

### 需要的支持

**资源请求**:
- 人力需求
- 预算需求
- 时间线
- 需要决策的问题

### 适用场景

| 场景 | 格式调整 |
|------|----------|
| 邮件发送 | 精简版，附件战略文档 |
| 会议简报 | 口头版，突出行动项 |
| 消息/IM | 超简版，一屏可看完 |
| 打印分发 | 完整版，保留所有细节 |

## 决策规则

| 条件 | 决策 |
|------|------|
| 受众为高管（VP及以上） | 侧重战略和ROI，篇幅≤1页，关键数据前置 |
| 受众为团队（IC/Manager） | 侧重执行和协作，篇幅≤2页，行动项明确 |
| 受众为外部（客户/合作伙伴） | 侧重价值和信任，篇幅≤1页，无敏感数据 |
| 简报字数>500字 | 自动精简至≤500字，标注"已精简" |
| 关键信息遗漏（战略目标/核心风险/行动项缺一） | 退回补充，不可发送 |
| 简报包含≥3个行动项 | 标记"行动项过多，建议聚焦Top3" |
| 敏感数据出现在外部简报中 | 自动脱敏或移除，标注"已脱敏" |

## 质量检查

- [ ] 一页能看完（邮件正文无需翻页）
- [ ] 内容精炼（每个模块1-3句话）
- [ ] 有行动指引（明确下一步）
- [ ] 格式适合目标渠道
- [ ] 关键信息不遗漏

## 降级策略

当上游文件不存在时，本Skill仍可独立执行：

| 缺失的上游文件 | 降级方案 |
|---------------|---------|
| strategy-doc.md | 用户提供战略要点 → 生成简报，标注"缺乏战略文档数据" |
| bmc.json / positioning-statement.json | 用户提供战略要点 → 生成简报，标注"缺乏业务和定位数据" |
| 所有上游文件均缺失 | 提示用户先执行前序阶段，或基于用户提供的战略要点生成简报 |

数据获取说明：
- 本Skill需要战略文档数据，请通过以下方式之一提供：
  1. 直接描述产品战略要点和核心信息
  2. 上传strategy-doc.md / bmc.json / positioning-statement.json文件
  3. 提供数据文件路径
- AI不负责外部数据采集，仅负责分析
