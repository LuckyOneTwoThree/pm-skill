# 隐私合规评估文档自动生成

## 核心原则

1. **默认隐私**——产品设计中默认采用最小数据采集和最短保留期
2. **合规是底线不是上限**——满足法规是最低要求，用户信任才是目标
3. **数据全生命周期覆盖**——从收集到删除，每个环节都必须合规
4. **可审计性**——每个合规判断必须有法规依据和证据支撑

## 执行步骤

### Step 1: 适用法规识别

基于目标市场识别适用的隐私法规：

| 地区 | 法规 | 关键要求 |
|------|------|---------|
| 中国 | PIPL（个人信息保护法） | 单独同意、数据本地化、个人信息影响评估 |
| 中国 | 网安法/数安法 | 等保、数据分类分级、跨境评估 |
| 欧盟 | GDPR | 合法依据、DPO、DPIA、72小时通知 |
| 美国 | CCPA/CPRA | 消费者权利、Do Not Sell |
| 全球 | COPPA | 13岁以下儿童数据保护 |

**法规适用矩阵**：

| 法规 | 是否适用 | 触发条件 | 合规等级 |
|------|---------|---------|---------|
| PIPL | | 处理中国境内个人信息 | 必须/建议/可选 |
| GDPR | | 处理欧盟居民数据 | 必须/建议/可选 |

| 匿名化数据 | 脱敏统计信息 | 无特殊要求 |

## 决策规则

| 条件 | 决策 |
|------|------|
| 目标市场包含中国 | 必须执行PIPL合规检查 |
| 目标市场包含欧盟 | 必须执行GDPR合规检查 |
| 涉及14岁以下用户 | 必须执行COPPA/儿童信息保护检查 |
| 涉及跨境数据传输 | 必须评估跨境传输合规 |
| PRD和API契约均缺失 | 基于用户提供信息盘点数据，标注"缺乏技术文档支撑" |

## 质量检查

- [ ] 适用法规已识别
- [ ] 个人信息盘点完整（含敏感信息标注）
- [ ] 合规检查清单逐项覆盖
- [ ] 每个差距有整改建议
- [ ] P0整改项已明确
- [ ] 第三方服务已评估
- [ ] 合规率已计算

## 降级策略

| 缺失的上游输入 | 降级方案 | 输出影响 |
|---------------|---------|---------|
| PRD缺失 | 基于用户提供的产品描述推导功能涉及的数据 | 数据盘点可能不完整 |
| API契约缺失 | 无法自动提取数据字段，需人工补充 | 数据映射可能不精确 |
| 数据模型缺失 | 无法自动识别数据存储和关系 | 数据流分析可能不完整 |
| 第三方清单缺失 | 跳过第三方评估章节 | 输出中标注"第三方评估待补充" |
| 已有隐私政策缺失 | 若用户未提供已有隐私政策，提示用户提供或跳过该输入相关步骤 | 隐私政策差距分析章节标注"待用户提供隐私政策" |
| 目标市场未指定 | 默认评估PIPL（中国），标注"建议确认目标市场" | 可能遗漏其他法规 |
