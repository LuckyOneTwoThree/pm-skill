# 模块7：产品增长与运营（pm-07-growth）

## 1. 定位

从用户获取到商业化的全链路增长引擎——以AARRR模型为骨架，通过数据驱动的渠道评估、激活优化、留存干预和商业化闭环，实现用户价值和商业价值的双赢增长。

## 2. 何时使用

- 需要评估获客渠道效果、优化获客漏斗时
- 需要识别Aha Moment、优化用户Onboarding路径时
- 需要构建流失预警模型、进行用户分层运营时
- 需要分析付费漏斗、追踪NRR、设计升级转化策略时
- 需要诊断产品增长模式、构建增长飞轮时

## 3. 子模块与编排器

| 子模块 | 编排器 | 作用 | 何时调用 |
|--------|--------|------|----------|
| 增长模式 | growth-orchestrator | 先诊断增长模式，再按需调度获客/激活/留存/变现子编排器，产出增长策略报告、GTM策略和运营手册 | 需要制定增长策略或系统化推进增长时 |
| 用户获取 | acquisition-orchestrator | 评估19种获客渠道、优化获客漏斗，让正确的用户找到产品 | 需要评估渠道ROI、优化获客转化时 |
| 用户激活 | activation-orchestrator | 识别Aha Moment、优化Onboarding路径，确保用户快速体验到核心价值 | 需要提升激活率、优化新用户引导时 |
| 用户留存 | retention-orchestrator | 构建流失预警模型、执行用户分层运营，解决留存和活跃问题 | 需要降低流失率、提升用户活跃度时 |
| 商业化 | revenue-orchestrator | 分析付费漏斗、追踪NRR、自动化升级转化，实现用户价值与商业价值双赢 | 需要提升付费转化率、优化收入结构时 |

## 4. Pipeline Skill 清单

### 4.1 增长模式（growth-orchestrator）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| growth-model | 分析产品特征、用户数据和商业模式，自动匹配最优增长模式，输出增长飞轮模型和瓶颈分析 | 产品特征、用户数据、商业模式 | 增长模式诊断（PLG/SLG/MLG/混合）、增长飞轮模型、关键约束条件、瓶颈分析 |
| growth-strategy-report | 将增长模式诊断和各环节优化方案汇总为完整可交付的增长策略报告，包含增长模式评估、AARRR漏斗诊断、杠杆策略、飞轮模型和执行路线图 | 增长模式诊断、获客方案、激活方案、留存方案、变现方案、业务目标 | 增长策略报告（含增长模式评估、AARRR漏斗诊断、杠杆策略、执行路线图） |
| gtm-strategy | 制定Go-to-Market策略，包含目标市场定义、上市路径选择、定价与包装策略、渠道与推广计划、上市里程碑与成功指标 | 产品定位、差异化策略、商业模式、定价方案、增长模式、产品信息 | GTM策略文档（含ICP画像、上市路径、定价包装、渠道计划、成功指标） |
| product-operations-manual | 将运营策略和流程汇总为完整可交付的产品运营手册，包含日常运营SOP、内容运营规范、用户运营策略、活动运营模板和应急响应流程 | 增长模式、激活策略、留存策略、变现策略、产品信息 | 产品运营手册（含日常SOP、内容运营规范、用户运营策略、活动模板、应急流程） |

### 4.2 用户获取（acquisition-orchestrator）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| acquisition-channel | 分析19种获客渠道数据，计算渠道规模、转化率、ROI，输出渠道分级报告 | 19种获客渠道数据、历史渠道表现数据、渠道配置和成本数据 | 渠道分级报告（主力/测试/观察渠道）、渠道ROI排名、资源分配建议 |
| acquisition-optimize | 分析获客漏斗数据，识别最大流失节点，自动生成优化方案和A/B测试设计 | 获客漏斗全链路数据、各渠道详细表现数据、历史优化实验数据 | 漏斗优化方案、流失节点分析、A/B测试设计方案 |

### 4.3 用户激活（activation-orchestrator）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| activation-aha | 分析留存数据和行为数据，自动识别Aha Moment候选，测量到达率，识别最短路径 | 留存数据（D1/D7/D30）、用户行为数据、用户分群数据 | Aha Moment候选列表、到达率分析、最短路径识别、Onboarding优化建议 |
| activation-onboarding | 分析Onboarding数据和用户分群，自动生成个性化引导策略，设计A/B测试方案 | Onboarding数据（完成率、流失率）、Aha Moment数据、用户分群数据 | 个性化Onboarding策略、分群引导路径、A/B测试设计方案 |

### 4.4 用户留存（retention-orchestrator）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| retention-churn | 构建流失预警模型，识别高风险用户，自动化触发干预动作，追踪干预效果 | 用户行为数据、流失历史数据、用户账户数据 | 流失预警模型、高风险用户列表、干预策略、干预效果追踪 |
| retention-engagement | 基于用户生命周期阶段进行分层，分析各层特征，自动生成运营策略和个性化触达内容 | 用户行为数据、用户生命周期阶段、用户账户数据 | 用户分层结果、各层特征分析、运营策略、个性化触达内容 |

### 4.5 商业化（revenue-orchestrator）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| revenue-funnel | 分析注册到付费全链路数据，识别付费障碍，计算转化优化建议，优化付费墙时机 | 注册到付费全链路数据、付费转化数据、用户特征数据 | 付费漏斗分析报告、付费障碍识别、付费墙优化建议、转化提升方案 |
| revenue-nrr | 自动计算净收入留存率，分析NRR趋势，识别流失预警和扩张收入机会 | 收入数据（MRR/ARR）、用户账户数据、用户行为数据 | NRR计算结果、趋势分析、流失预警、扩张收入机会识别 |
| revenue-upsell | 识别升级信号用户，自动生成个性化升级内容，优化触达时机，设计A/B测试 | 用户行为数据、付费历史数据、产品使用数据 | 升级信号用户列表、个性化升级内容、触达时机优化、A/B测试设计方案 |

## 9. 核心信念

| 子模块 | 核心信念 |
|--------|----------|
| 增长模式 | 增长飞轮的起点是诊断而非行动——先理解增长模式，再选择增长杠杆 |
| 用户获取 | 让正确的用户找到产品——用户获取不是流量游戏，而是匹配游戏 |
| 用户激活 | Aha Moment是用户留存的起点——没有Aha Moment的激活只是流程完成，不是价值传递 |
| 用户留存 | 留存是衡量产品价值的核心指标——留存问题的根因永远是价值问题，而非运营问题 |
| 商业化 | 商业化是用户价值和商业价值的双赢——好的商业化不是从用户身上榨取价值 |
