# 模块9：项目管理与执行（贯穿全程）

## 定位

贯穿产品全生命周期的执行保障层——从项目规划启动、敏捷迭代执行到风险管控，确保每个阶段有序推进、风险可控。

## 何时使用

- 产品项目启动，需要制定项目宪章与资源规划
- 进入执行阶段，需要Sprint规划与每日同步
- 项目运行中出现风险信号，需要识别、监控与升级
- 任何阶段需要项目管理框架支撑时（本模块贯穿全程）

## 子模块与编排器

| 子模块 | 编排器 | 作用 | 何时调用 |
|--------|--------|------|----------|
| 项目规划与启动 | project-planning-orchestrator | 制定项目宪章、锁定资源、召开Kickoff，为项目奠定有序基础 | 项目启动阶段，或重大阶段转换需重新规划时 |
| 敏捷执行 | agile-orchestrator | Sprint规划、每日同步、Sprint评审，保障迭代节奏与交付质量 | 项目进入执行阶段后，每个Sprint周期持续运行 |
| 风险管理 | risk-orchestrator | 识别风险、持续监控、触发升级，确保风险始终受控 | 项目全程持续运行，风险信号出现时立即响应 |

## Pipeline Skill 清单

### 项目规划与启动

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| planning-project-charter | 生成项目宪章，明确项目目标、范围、成功标准与里程碑 | 产品规划文档、业务目标、干系人需求 | 项目宪章（含目标、范围、成功标准、里程碑） |
| planning-resource | 评估资源需求与可用资源，制定资源分配与预留方案 | 项目宪章、团队产能、资源日历 | 资源需求清单、资源分配方案、关键资源预留确认 |
| planning-kickoff | 筹备并执行Kickoff会议，对齐目标并提取行动项 | 项目宪章、资源分配方案、干系人名单 | Kickoff会议记录、行动项清单、干系人共识确认 |

### 敏捷执行

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| agile-sprint-planning | 制定Sprint计划，定义Sprint Goal、分配Story、确认容量 | 产品Backlog、团队容量、迭代目标 | Sprint Goal、Story分配方案、容量确认、Sprint Backlog |
| agile-daily-sync | 每日进度同步，识别障碍并标记，重大障碍触发升级 | Sprint Backlog、团队成员状态更新 | 每日同步记录、障碍清单、升级标记 |
| agile-review | Sprint评审与回顾，评估目标达成情况并沉淀改进 | Sprint执行数据、交付成果、障碍记录 | Sprint评审报告、目标达成率、改进建议 |
| sprint-retrospective-report | 将Sprint执行数据汇总为完整可交付的复盘报告，包含Sprint目标达成分析、交付质量评估、团队速率趋势、改进行动项和下一Sprint建议 | Sprint计划、每日同步记录、Sprint评审结果、历史Sprint数据 | Sprint复盘报告（含目标达成分析、交付质量、速率趋势、改进行动项） |

### 风险管理

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| risk-identification | 识别项目风险，评估概率与影响，建立风险登记册 | 项目文档、历史风险数据、干系人输入 | 风险登记册（含风险描述、概率、影响、优先级） |
| risk-monitoring | 持续监控风险状态，检测风险指标变化与触发条件 | 风险登记册、项目执行数据、风险指标 | 风险状态更新报告、触发预警通知 |
| risk-escalation | 高优先级风险升级处理，启动应对流程与资源调配 | 风险预警通知、升级规则 | 升级通知、应对方案、资源调配请求 |

## 阶段卡口

| 卡口 | 通过标准 |
|------|----------|
| 宪章已批准 | 项目宪章经人类审批，目标和范围已确认 |
| 资源已锁定 | 资源需求与可用资源匹配，关键资源已预留 |
| Kickoff已完成 | Kickoff会议已召开，行动项已提取 |
| Sprint计划已确认 | Sprint Goal已定义，Story已分配，容量已确认 |
| Daily Sync障碍已暴露 | 每日障碍已识别并标记，重大障碍已升级 |
| 风险登记册已建立 | 风险识别完成，风险登记册已生成且持续更新 |
| 高优先级风险已升级 | Critical/High风险已触发升级流程 |

## 人类决策点

| 决策点 | 触发条件 | 决策内容 |
|--------|----------|----------|
| 宪章审批 | 项目宪章生成完成时 | 审批项目目标、范围和成功标准 |
| 范围变更 | 执行过程中出现范围变更请求时 | 评估变更影响，决定是否接受变更 |
| Sprint Goal确认 | Sprint Planning完成时 | 确认Sprint目标的合理性和可达性 |
| Sprint取消决策 | Sprint目标连续未达成或重大范围变更时 | 决定是否取消当前Sprint |
| 风险应对策略确认 | 新风险识别或风险状态变化时 | 确认应对策略（规避/转移/减轻/接受） |
| 升级决策 | 风险升级触发时 | 确认升级路径和资源调配 |

## 核心信念

- **项目规划与启动**：好的开始是成功的一半，混乱的开始无法补救
- **敏捷执行**：Sprint是节奏不是目标，交付是结果不是目的
- **风险管理**：风险不会自己消失，你不管理它它就管理你
