# 模块8：产品监控与迭代

## 定位

产品上线后的持续守护层——实时监控产品运行状态，智能诊断问题根因，驱动数据导向的迭代决策。

## 何时使用

- 产品已上线运行，需要建立持续监控体系
- 出现异常指标或用户反馈，需要快速定位根因
- 积累了一定运行数据，需要基于数据制定迭代优先级
- 竞品发生功能变更，需要评估影响并调整策略

## 子模块与编排器

| 子模块 | 编排器 | 作用 | 何时调用 |
|--------|--------|------|----------|
| 监控预警 | monitoring-orchestrator | 建立监控体系，实时检测异常并触发告警 | 产品上线后持续运行，或需要新建/调整监控覆盖时 |
| 智能诊断 | diagnosis-orchestrator | 对异常信号进行健康度评估与竞品分析，定位问题根因 | 监控预警触发异常告警后，或定期健康巡检时 |
| 迭代决策 | iteration-orchestrator | 整理问题Backlog，评估优先级，生成迭代调整方案 | 诊断完成需制定行动方案时，或迭代周期规划节点 |

## Pipeline Skill 清单

### 监控预警

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| monitoring-pipeline | 监控预警全流程：建立核心路径监控体系、实时检测指标异常、构建可视化监控看板、告警分级与升级处理 | 产品架构、指标体系、SLA要求、用户角色、On-Call排班 | 监控配置方案、异常事件列表、监控看板、升级通知、值班报告 |

### 智能诊断

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| diagnosis-health | 评估产品整体健康度，计算多维度健康评分 | 监控指标数据、用户反馈数据 | 健康度评分报告、各维度评分明细 |
| diagnosis-competition | 追踪竞品动态变更，分析竞品优劣势与市场影响 | 竞品信息源、产品功能清单 | 竞品动态报告、优劣势对比分析 |

### 迭代决策

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| iteration-decision | 迭代决策全流程：Backlog整理与优化、优先级评估与调整方案、迭代回顾与改进建议 | 问题列表、健康度报告、竞品分析、迭代数据 | 优化后Backlog、优先级排序、变更影响评估、调整方案、回顾报告、改进建议 |

### 报告

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| competitor-monitoring-report | 将竞品追踪数据汇总为完整可交付的监控报告，包含竞品动态汇总、功能变更追踪、市场策略变化、威胁评估和应对建议 | 竞品追踪数据、竞品情报、竞品分类、监控周期 | 竞品动态监控报告（含动态汇总、功能变更、威胁评估、应对建议） |
| user-feedback-loop-report | 将用户反馈处理过程汇总为完整可交付的闭环报告，包含反馈来源分析、处理进度追踪、闭环率统计、未解决问题和改进建议 | 用户声音分析、异常监控数据、反馈数据、处理记录 | 用户反馈闭环报告（含来源分析、闭环率统计、未解决问题、改进建议） |

### 质量与发布

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| quality-acceptance | 质量验收：基于验收标准清单逐项验证，生成验收报告与风险项清单 | 验收标准、测试报告、上线检查清单 | 质量验收报告（含验收结果、风险项、放行建议） |
| release-gradual | 灰度发布：制定分阶段发布策略，配置灰度规则与回滚条件 | 发布计划、灰度策略、监控配置 | 灰度发布方案（含阶段配置、流量规则、回滚条件） |
| release-auto-checklist | 发布自动检查：自动生成发布前检查清单，逐项验证并输出检查结果 | 发布内容、环境配置、依赖清单 | 发布检查报告（含检查项、通过状态、风险提示） |
| release-notes | 发布说明：自动生成面向不同受众的发布说明文档 | 发布内容、变更记录、用户影响 | 发布说明文档（含用户版、运维版、内部版） |

### 退出

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| product-sunset-plan | 制定产品或功能下线计划，包含下线决策评估、用户迁移方案、数据处置策略、时间线和沟通计划 | 健康度诊断、留存数据、下线对象、下线原因 | 产品下线方案（含决策评估、迁移方案、数据处置、时间线、沟通计划） |

## 阶段卡口

| 卡口 | 通过标准 |
|------|----------|
| 核心路径覆盖率 | ≥95%，产品核心路径已全部纳入监控 |
| 告警噪音率 | <15%，告警中误报和低价值告警占比低于15% |
| 健康度评分偏差 | ±10%以内，健康度评分与实际状态偏差在可控范围内 |
| 竞品动态追踪 | 竞品功能变更已识别，优劣势分析已完成 |
| 迭代决策完成 | Backlog已优化，优先级调整方案已生成，迭代复盘完成且改进建议经人类确认 |
| 竞品监控报告已生成 | 竞品动态已汇总，威胁评估和应对建议已输出 |
| 反馈闭环报告已生成 | 反馈闭环率已统计，未解决问题和改进建议已输出 |
| 质量验收已完成 | 验收标准逐项验证完成，风险项已列出，放行建议已生成 |
| 灰度发布方案已确认 | 灰度阶段配置完成，流量规则与回滚条件经人类确认 |
| 发布检查已通过 | 发布前检查清单逐项验证完成，所有阻断项已解决 |
| 发布说明已生成 | 面向不同受众的发布说明文档已生成 |
| 下线方案已确认 | 下线决策评估完成，用户迁移和数据处置方案经人类确认 |

## 人类决策点

| 决策点 | 触发条件 | 决策内容 |
|--------|----------|----------|
| 告警阈值调整 | 告警噪音率过高或漏报率过高时 | 确认告警阈值调整方案 |
| 健康度评分校准 | 健康度评分与实际感知偏差超过±10%时 | 确认评分模型校准方案和权重调整 |
| 迭代计划调整确认 | 优先级调整方案生成完成时 | 确认调整方案、资源重新分配和风险接受 |
| 竞品应对策略确认 | 竞品监控报告生成完成时 | 确认威胁评估和应对策略 |
| 质量验收放行决策 | 质量验收报告生成完成时 | 确认验收结果，决定是否放行或附加条件放行 |
| 灰度发布策略确认 | 灰度发布方案生成完成时 | 确认灰度阶段、流量规则和回滚条件 |
| 下线决策确认 | 产品下线方案生成完成时 | 确认下线决策、用户迁移方案和数据处置策略 |

## 核心信念

- **监控预警**：让问题在用户发现之前被解决
- **智能诊断**：快速定位问题根因，减少MTTR
- **迭代决策**：数据驱动的优先级决策，平衡短期修复与长期价值
- **报告**：将零散数据转化为结构化洞察，报告是决策的弹药
- **质量与发布**：发布是产品价值的最后一公里，质量是底线不是目标
- **退出**：下线是对用户最后的尊重，有序过渡胜过突然消失
